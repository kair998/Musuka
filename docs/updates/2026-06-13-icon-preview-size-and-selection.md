# 2026-06-13：图标预览、尺寸与框选反馈修复

## 概述

本次更新修复设置页原生图标预览颠倒、桌面部分原生图标显示极小，以及左键拖动群体框选反馈不明显的问题。

## 图标预览

- 修正 GDI+ 底向上位图转换到 Qt 图像后的垂直方向。
- 原生图标预览不再上下颠倒。
- 原生图标预览与桌面渲染共用异常透明画布处理逻辑。

## 桌面图标尺寸

- 对原生图标进行条件透明画布裁边。
- 仅当可见内容相对画布异常小时才裁边，正常原生图标和用户导入图片不受影响。
- 自动排列只调整图标位置，不再为寻找空位逐级缩小并保存图标尺寸。
- 已有配置中的图标尺寸在自动排列后保持不变。

## 群体框选

- 左键拖动框选过程中实时更新已选对象。
- 提高框选区域填充和边框的可见度。
- 为已选对象增加独立的蓝色高亮背景和边框。

## 测试

- 新增异常透明画布裁边回归测试。
- 新增自动排列保持配置图标尺寸的回归测试。
- `build.bat`：通过。
- `ctest --test-dir build-nmake --output-on-failure`：通过，1/1 测试通过。
- `git diff --check`：通过。
