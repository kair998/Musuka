# Musuka 性能优化报告

日期：2026-06-14

## 优化范围

本轮优化覆盖发布包与构建目录空间、默认图片资产、设置页图片加载、桌面图标资源复用、拖动与框选热路径、配置保存频率，以及应用启动生命周期。

测量环境为 Windows x64、Qt 6.8.3、MSVC 2022、NMake Release 构建。目录大小按目录内全部文件字节数统计。

## 空间结果

| 项目 | 优化前 | 优化后 | 减少 |
|---|---:|---:|---:|
| `release` 发布包 | 87.23 MiB / 196 文件 | 35.12 MiB / 36 文件 | 52.11 MiB，59.7% |
| `build-nmake` 构建目录 | 124.09 MiB / 512 文件 | 51.50 MiB / 491 文件 | 72.59 MiB，58.5% |
| `default_image` | 15.52 MiB | 10.42 MiB | 5.10 MiB，32.9% |
| 发布包 `data` | 4.12 MiB / 140 文件 | 0 MiB / 0 文件 | 不再携带本地用户数据 |

主要措施：

- 将 10 张超大内置 PNG 的最长边限制为 1024 像素，保留透明通道。
- `windeployqt` 排除软件 OpenGL、D3D/DXC 编译器、未使用的 Qt Network/SVG 运行库和无关插件。
- 保留程序实际需要的 Qt Core/Gui/Widgets、Windows 平台插件、JPEG 插件和 MSVC CRT。
- 打包始终创建空 `release\data`，不再复制开发机本地配置和图标数据。
- 构建脚本主动清理旧部署遗留文件，避免增量构建目录持续膨胀。

## 运行时优化

### 图标资源复用

桌面图标层按“规范化图片路径 + 原始图标/替换图类型”缓存已解码、裁剪和预处理的位图。多个对象使用同一图片时共享一个 `Gdiplus::Bitmap`，失败的加载结果也会缓存，避免重复尝试。

自动化测试中的 8 个对象均使用同一张图片，优化后只保留 1 份位图。以 1254 x 1254、32 位图片为例，解码内存可由约 48 MiB 降至约 6 MiB，减少约 87.5%。

### 拖动、缩放与框选

- 建立对象索引到渲染项索引的映射，将对象查找由线性扫描改为 O(1)。
- 拖动时只重新计算被拖动项的矩形，不再重算所有图标。
- 滚轮缩放时只重新计算选中项。
- 框选只失效旧/新选择框和选择状态变化的图标，不再让整个桌面窗口重绘。

因此，拖动和缩放的核心几何更新由依赖全部渲染项数量，收敛为主要依赖当前选中项数量。

### 设置页图片加载

- 缩略图使用 `QImageReader::setScaledSize` 在解码阶段缩放，避免先完整解码大图再缩小。
- 加载前限制文件大小、单边尺寸和总像素数。
- 使用包含路径、文件大小、修改时间和目标尺寸的 `QPixmapCache` 键；重复显示同一缩略图时直接复用。
- 去除“先完整验证解码、再由 Qt 再次解码”的双重加载路径。

### 配置保存与启动

- 高频设置修改使用 250 ms 防抖保存，并在关闭、切换语言和运行桌面前强制刷新。
- `QApplication` 使用 CRT 提供的真实 `__argc/__argv`，不再构造短生命周期伪参数。
- 设置窗口由主函数按 Qt 生命周期明确持有，`App` 只保存非拥有指针并协调显示，避免顶层 Qt 窗口的动态所有权异常。
- 删除设置窗口启动时重复的 Win32 图标转 Qt 图片转换；EXE 内嵌图标仍保留。
- 中文长说明使用纯文本和显式换行，避开不必要的富文本/CJK 自动布局开销。

## 验证

- `ctest --test-dir build-nmake --output-on-failure`：通过。
- smoke 测试覆盖设置窗口挂接、多语言切换、两种兼容模式、共享位图复用和 Explorer 图标恢复。
- smoke 测试连续运行 20 次：全部通过。
- 正式 `release\musuka.exe` 连续启动驻留测试：10 次均保持运行，无启动期堆破坏。
- 发布包确认仅包含所需 Qt/CRT 文件，`release\data` 为空。
- `git diff --check`：通过。

## 限制与后续测量

本轮已量化磁盘占用和共享位图数量，并通过结构性测试验证热路径变化；尚未引入独立的帧时间、输入延迟和峰值工作集基准工具。因此不对拖动帧率或启动毫秒数给出未经测量的百分比结论。

后续如需持续跟踪，可增加包含 100、500、1000 个对象的拖动基准，记录 P50/P95 消息处理时间、首次设置页显示时间和进程峰值工作集。
